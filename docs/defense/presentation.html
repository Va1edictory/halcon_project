项目背景与意义

🌐 工业背景

晶体管作为集成电路的核心基础元件，其质量直接决定电子产品的可靠性和使用寿命。随着消费电子、汽车电子、工业控制等领域对元器件质量要求的不断提高，晶体管外观缺陷检测已成为关键的质量控制环节。

主要挑战：
  • 🔧 缺陷多样性：引脚弯曲、切割不良、外壳破损、元件错位等多种缺陷
  • 📦 生产规模庞大：单条生产线日产量可达数十万甚至数百万件
  • 🎯 检测标准严苛：微小缺陷可能导致整个电路系统失效
  • 💰 成本压力：人工误检漏检可能引发召回风险和品牌损害

✕ 人工目检的痛点

  • 🔴 效率低 — 1-2 件/秒，难以满足大规模生产
  • 🔴 主观性强 — 不同质检员标准不一
  • 🔴 易疲劳 — 长时间工作漏检率上升
  • 🔴 难追溯 — 无法系统记录缺陷数据

✓ 自动检测的优势

  • 🟢 效率提升 5-10 倍 — 10 件/秒以上
  • 🟢 基于模型的标准化判定
  • 🟢 检测一致性高，不受疲劳影响
  • 🟢 结果数字化，便于统计与追溯

📋 应用场景

  • 🏭 生产线在线检测：集成到流水线，实时自动检测与分选
  • 📦 来料检验：电子制造企业对采购晶体管进行批量抽检或全检
  • 🔍 质量追溯分析：精确定位缺陷位置，辅助工艺优化
  • 🔄 自动化产线升级：提供"机器换人"解决方案
工程应用价值：
✅ 验证了 GC Anomaly Detection 模型在晶体管缺陷检测中的有效性，解决缺陷样本稀缺痛点
✅ 实现了完整检测流水线，手动五步预处理具有技术参考价值
✅ 支持单张/批量两种模式，架构灵活可扩展
✅ 基于 HALCON 平台，工业部署友好，成本效益优势明显
核心目标：利用 HALCON DLT 深度学习平台，构建一套能自动判断晶体管 OK/NG 并标注缺陷位置的非接触式检测系统。

技术流水线详解

① 图像预处理

背景：HALCON 26 中移除了 preprocess_dl_model 算子，必须手动实现完整的预处理流程。
原始图像 (900×900, byte, 1ch/3ch)
    │
    ├─ ① 通道转换 ──► 3ch 图像
    │
    ├─ ② 尺寸缩放 ──► 256×256
    │
    ├─ ③ 类型转换 ──► real 类型
    │
    ├─ ④ 值域映射 ──► [-127, 128]
    │
    ├─ ⑤ 域的设置 ──► full_domain
    │
    ▼
DLSample（模型推理输入）
步骤算子变换原因
① 通道转换count_channels + compose31ch → 3ch模型要求 3 通道输入
② 尺寸缩放zoom_image_size900² → 256²模型输入固定 256×256
③ 类型转换convert_image_typebyte → real模型需要浮点类型
④ 值域映射scale_image(x,1.0,-127)[0,255] → [-127,128]匹配训练时的值域
⑤ 域的设置full_domain—避免 domain 裁剪影响推理
⑥ 样本生成gen_dl_samples_from_images→ DLSample避免 iconic/control 类型冲突
* ============ 手动五步预处理流水线 ============ * ① 通道转换：确保 3 通道 count_channels (Image, NumChannels) if (NumChannels == 1) compose3 (Image, Image, Image, Image3Ch) else Image3Ch := Image endif * ② 尺寸缩放：900×900 → 256×256 zoom_image_size (Image3Ch, ImageScaled, 256, 256, 'constant') * ③ 类型转换：byte → real convert_image_type (ImageScaled, ImageReal, 'real') * ④ 值域映射：[0,255] → [-127,128] scale_image (ImageReal, ImageScaledVal, 1.0, -127) * ⑤ 域的设置：full_domain full_domain (ImageScaledVal, ImageFullDomain) * ⑥ 样本生成：创建 DLSample gen_dl_samples_from_images (ImageFullDomain, [], [], DLPreprocessParams, DLSample)

关键技术难点与解决方案

难点问题描述解决方案
Iconic/Control 类型冲突 set_dict_tuple 不接收 iconic 图像变量 改用 gen_dl_samples_from_images
异常图提取失败 get_dict_tuple 返回 control 值 改用 get_dict_object
preprocess_dl_model 缺失 HALCON 26 已移除此算子 手动实现 5 步预处理流水线
坐标偏差 256² vs 900² 尺寸不匹配 仿射变换 affine_trans_region
文本不可见 disp_message 坐标系错误 'window' → 'image' 坐标系
通道不匹配 原图 3ch vs 热力图 1ch rgb1_to_gray 统一为单通道
批量变量未定义 ImgWidth/ImgHeight 批量循环中未初始化 每张图片单独 get_image_size
